图与树递归特训

目标不是背几段答案,而是形成一套稳定的递归分析动作:先看当前节点或当前顶点,再决定对子结构递归,最后把局部信息和递归结果合并。

训练顺序
  1. 从实例模拟递归调用
  2. 填空补齐关键结构
  3. 先规划变量和返回值
  4. 独立写完整伪代码
递归分析法 树:从例子开始 树:填空训练 图:visited 思维 结构规划 完整伪代码练习 高频错误

一、先固定递归的四个问题

遇到 arbre / graphe / recursivite 题,不要先急着写循环或代码。先把下面四件事写出来。

1. 当前对象是什么?

树题通常是一个节点 N;图题通常是一个顶点 s,再配合图 G 和已访问集合 M。

2. 当前对象自己贡献什么?

例如当前节点是否满足条件、当前节点标签是多少、当前顶点是否需要标记。

3. 对哪些更小结构递归?

树是对子节点 enfants(N);图是对未访问邻居 voisins(G, s)。

4. 怎么合并返回值?

计数用加法,最大值用 max,存在性用“只要一个为真”,路径用“找到后接上当前节点”。

考试版模板 
fonction f(N)
// noeud -> type_de_retour
resultat <- contribution_du_noeud_N
pour tout enfant E de N
  resultat <- combiner(resultat, f(E))
retourner resultat

这就是绝大多数树递归题的骨架。难点不在“递归”这两个字,而在 contribution 和 combiner 怎么定义。

二、树递归:从手算实例开始

r / | \ a b e / \ c d

先用一棵小树练习。这里 r 有 3 个孩子,b 有 2 个孩子,a,c,d,e 是叶子。

问题当前节点贡献合并方式
节点总数 taille当前节点贡献 11 + 所有子树大小
高度 hauteur叶子高度为 11 + 子树高度最大值
叶子数叶子贡献 1,内部节点贡献 0所有子树叶子数相加

例 1:节点总数

fonction taille(N)
// noeud -> entier
total <- 1
pour tout enfant E de N
  total <- total + taille(E)
retourner total

这段伪代码的意思是:每个节点先数自己,再让每个孩子去数自己的子树。

例 2:高度

fonction hauteur(N)
// noeud -> entier
si N est une feuille
  retourner 1
h <- 0
pour tout enfant E de N
  h <- max(h, hauteur(E))
retourner h + 1

高度不是把所有子树高度相加,而是取最深的那一条路,所以用 max。

三、树递归填空:从骨架到完整函数

练习 A:统计叶子数

补全伪代码。

fonction nb_feuilles(N)
// noeud -> entier
si N est une feuille
  retourner ____
total <- ____
pour tout enfant E de N
  total <- total + ____
retourner total
查看答案
fonction nb_feuilles(N)
// noeud -> entier
si N est une feuille
  retourner 1
total <- 0
pour tout enfant E de N
  total <- total + nb_feuilles(E)
retourner total

叶子自己就是一个叶子;内部节点只负责把各子树结果加起来。

练习 B:统计“有叶子孩子”的节点数

这类题非常像真题。注意计数对象是“父节点”,不是叶子。

fonction nb_avec_feuille(N)
// noeud -> entier
total <- 0
a_feuille <- faux
pour tout enfant E de N
  si E est une feuille
    a_feuille <- ____
  total <- total + ____
si a_feuille
  total <- ____
retourner total
查看答案
fonction nb_avec_feuille(N)
// noeud -> entier
total <- 0
a_feuille <- faux
pour tout enfant E de N
  si E est une feuille
    a_feuille <- vrai
  total <- total + nb_avec_feuille(E)
si a_feuille
  total <- total + 1
retourner total

先看当前节点是否有叶子孩子,同时递归累计每个孩子子树中的答案。

练习 C:最大根叶路径和

节点标签都是数。求从当前节点到某个叶子的路径上,标签和的最大值。

fonction somme_max(N)
// noeud -> nombre
si N est une feuille
  retourner etiquette(N)
m <- -infini
pour tout enfant E de N
  m <- max(m, ____)
retourner ____ + m
查看答案
fonction somme_max(N)
// noeud -> nombre
si N est une feuille
  retourner etiquette(N)
m <- -infini
pour tout enfant E de N
  m <- max(m, somme_max(E))
retourner etiquette(N) + m

路径只能选择一个孩子继续往下,所以合并方式是 max,不是求和。

四、图递归:先建立 visited 思维

A --- B --- D | | C --- E

图和树最大的不同:图可能有环。递归访问图时,如果没有已访问集合 M,可能在 A - B - A - B 之间无限循环。

图递归基本规则

进入顶点 s 后,立刻把 s 标记为已访问;之后只递归访问 t ∉ M 的邻居。

例 3:DFS 探索一个连通分量

fonction explore(G, s, M)
// graphe x sommet x ensemble(modifie) ->
ajouter s a M
pour tout voisin t de s dans G
  si t n'appartient pas a M
    explore(G, t, M)

M 是会被修改的集合。调用结束后,M 中包含从 s 能到达的所有顶点。

练习 D:可达性

补全递归函数。它判断从 s 是否能到达 cible。

fonction accessible(G, s, cible, M)
// graphe x sommet x sommet x ensemble -> booleen
si s = cible
  retourner ____
ajouter s a M
pour tout voisin t de s
  si t n'appartient pas a M
    si accessible(G, t, cible, M)
      retourner ____
retourner ____
查看答案
fonction accessible(G, s, cible, M)
// graphe x sommet x sommet x ensemble -> booleen
si s = cible
  retourner vrai
ajouter s a M
pour tout voisin t de s
  si t n'appartient pas a M
    si accessible(G, t, cible, M)
      retourner vrai
retourner faux

存在性问题用“找到一个成功分支就立刻返回 vrai”。所有分支失败才返回 faux。

五、正式写之前,先做结构规划

结构规划不是额外步骤,它是防止递归题写乱的核心。下面这张表可以直接当草稿纸模板。

规划项树题怎么填图题怎么填
输入类型noeud 或 arbregraphe x sommet,常加 ensemble
输出类型entier, booleen, liste, nombrebooleen, entier, 或修改集合
基准情形叶子、空孩子列表、标签满足目标已到目标、没有新邻居、顶点已访问
局部贡献当前节点是否计数、当前标签、当前 score标记当前顶点、当前顶点是否计数
递归对象每个孩子 E每个未访问邻居 t
合并方式加法、最大值、最小值、逻辑或、路径拼接加法、逻辑或、修改已访问集合

示范:从题目到伪代码

题目

函数 f(N) 输入一棵标签为数的树,返回“所有节点中,其孩子标签和”的最大值。叶子孩子标签和按 0 计。

输出最大值,所以返回 nombre
当前贡献当前节点的孩子标签和 s
递归对象每个孩子 E
合并方式max(当前贡献, 每个子树最大值)
fonction f(N)
// noeud -> nombre
s <- 0
m <- 0
pour tout enfant E de N
  s <- s + etiquette(E)
  m <- max(m, f(E))
retourner max(s, m)

六、完整伪代码练习

这一组按难度递增。建议先只写结构规划,再写完整伪代码,最后打开答案对比。

  1. 练习 1:树的最小标签

    写函数 min_etiquette(N),返回树中所有节点标签的最小值。

    参考答案
    fonction min_etiquette(N)
// noeud -> nombre
m <- etiquette(N)
pour tout enfant E de N
  m <- min(m, min_etiquette(E))
retourner m
  2. 练习 2:树中恰有两个孩子的节点数

    写函数 deux_enfants(N),返回整棵树中孩子数恰好为 2 的节点个数。

    参考答案
    fonction deux_enfants(N)
// noeud -> entier
compte <- 0
si nombre_enfants(N) = 2
  compte <- 1
pour tout enfant E de N
  compte <- compte + deux_enfants(E)
retourner compte
  3. 练习 3:树中是否存在标签 x

    写函数 contient(N, x),若树中有某个节点标签等于 x,返回 vrai,否则返回 faux。

    参考答案
    fonction contient(N, x)
// noeud x etiquette -> booleen
si etiquette(N) = x
  retourner vrai
pour tout enfant E de N
  si contient(E, x)
    retourner vrai
retourner faux
  4. 练习 4:从根到目标的路径

    写函数 chemin(N, x)。若存在标签 x,返回从根到该节点的标签列表;否则返回空列表。

    参考答案
    fonction chemin(N, x)
// noeud x etiquette -> liste
si etiquette(N) = x
  retourner [x]
pour tout enfant E de N
  C <- chemin(E, x)
  si C n'est pas vide
    retourner [etiquette(N)] + C
retourner []

    这是“存在性 + 返回结构”的题。找到子树路径后,把当前根接到前面。

  5. 练习 5:图中可达顶点数

    写递归过程 explore(G, s, M),使调用结束后 M 包含从 s 可达的所有顶点。再写函数 nb_accessibles(G, s) 返回可达顶点个数。

    参考答案
    fonction explore(G, s, M)
// graphe x sommet x ensemble(modifie) ->
ajouter s a M
pour tout voisin t de s
  si t n'appartient pas a M
    explore(G, t, M)

fonction nb_accessibles(G, s)
// graphe x sommet -> entier
M <- ensemble_vide()
explore(G, s, M)
retourner cardinal(M)
  6. 练习 6:图的连通分量数

    写函数 nb_composantes(G),返回无向图的连通分量个数。

    参考答案
    fonction nb_composantes(G)
// graphe -> entier
M <- ensemble_vide()
n <- 0
pour tout sommet s de G
  si s n'appartient pas a M
    n <- n + 1
    explore(G, s, M)
retourner n

    每遇到一个还没访问过的顶点,就发现了一个新分量,然后用 DFS 一口气标记整个分量。

  7. 练习 7:有限步可达

    写函数 chemin_borne(G, s, t, k, M),判断是否存在从 s 到 t 且长度不超过 k 的路径。可以假设路径不重复顶点。

    参考答案
    fonction chemin_borne(G, s, t, k, M)
// graphe x sommet x sommet x entier x ensemble -> booleen
si s = t
  retourner vrai
si k = 0
  retourner faux
ajouter s a M
pour tout voisin u de s
  si u n'appartient pas a M
    si chemin_borne(G, u, t, k-1, M)
      retourner vrai
retourner faux

    这里 k 是递归规模。每走一条边就把 k 减 1。

  8. 练习 8:树到图的思维切换

    解释为什么树递归通常不需要 M,而图递归通常需要 M。再写一句话说明 M 应该在什么时候更新。

    参考答案

    树天然没有回到父节点的环,每个节点只有唯一父节点,所以沿孩子递归不会无限绕圈。图可能有环,例如 A-B-A,因此必须记录访问过的顶点。

    进入顶点后应尽早把它加入 M,通常是在递归函数开始处、遍历邻居之前。

七、递归题高频错误清单

  • 把“当前节点的贡献”漏掉,只返回子树结果。
  • 在 for 循环第一轮就 retourner,导致只处理第一个孩子。
  • 该用 max 的题误用求和,尤其是高度和路径最优值。
  • 该用求和的题误用 max,尤其是节点计数。
  • 图递归忘记 M,或者标记太晚,导致重复访问。
  • 把“邻居”与“所有可达顶点”混淆。
  • 伪代码没写 signature,被扣掉容易拿的分。
  • 在图题里修改 G 本身,而题目只要求遍历。
  • 路径题找到子路径后忘记把当前节点接在前面。
  • 布尔存在性题所有分支失败后忘记返回 faux。
最后的自检句

写完后问自己:如果当前节点是叶子会怎样? 如果当前节点有 3 个孩子会不会全处理? 如果图里有环会不会无限递归? 返回值类型是不是和 signature 一致?
