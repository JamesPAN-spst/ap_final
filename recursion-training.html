图与树递归特训

目标不是死记几段答案,而是形成一套稳定的递归分析动作:先看当前节点或当前顶点,再决定对子结构递归,最后把局部信息和递归结果合并起来。

训练顺序
  1. 从实例模拟递归调用
  2. 填空补齐关键结构
  3. 先规划变量和返回值
  4. 独立写完整伪代码
递归分析法 树：从例子开始 树：填空训练 图：已访问集合思维 结构规划 完整伪代码练习 高频错误

一、先固定递归的四个问题

遇到 树 / 图 / 递归 题时,不要先急着写循环或代码。先把下面四件事写出来。

1. 当前对象是什么？

树题通常处理一个节点 N; 图题通常处理一个顶点 s,再配合图 G 和已访问集合 M。

2. 当前对象自己贡献什么？

例如当前节点是否满足条件、当前节点标签是多少、当前顶点是否需要标记。

3. 对哪些更小的结构递归？

树题是对子节点 子节点(N) 递归; 图题是对尚未访问的邻居 邻居(G, s) 递归。

4. 怎么合并返回值？

计数用加法,最大值用 max,存在性用“只要一个分支为真即可”,路径题则是“找到后接上当前节点”。

考试版模板 
函数 f(N)
// 节点 -> 返回类型
结果 <- 当前节点的贡献
对 N 的每个子节点 E
  结果 <- 合并(结果, f(E))
返回 结果

这就是绝大多数树递归题的骨架。难点不在“递归”这两个字,而在“当前贡献”和“如何合并”到底怎么定义。

二、树递归：从手算实例开始

r / | \ a b e / \ c d

先用一棵小树练习。这里 r 有 3 个子节点,b 有 2 个子节点,a,c,d,e 都是叶子。

问题当前节点贡献合并方式
节点总数（taille）当前节点贡献 11 + 所有子树的大小
高度（hauteur）叶子高度记为 11 + 子树高度的最大值
叶子数叶子贡献 1,内部节点贡献 0所有子树叶子数相加

例 1：节点总数

函数 节点总数(N)
// 节点 -> 整数
总数 <- 1
对 N 的每个子节点 E
  总数 <- 总数 + 节点总数(E)
返回 总数

这段伪代码的意思是：每个节点先数自己,再让每个孩子去统计自己的子树。

例 2：高度

函数 高度(N)
// 节点 -> 整数
如果 N 是叶子
  返回 1
h <- 0
对 N 的每个子节点 E
  h <- max(h, 高度(E))
返回 h + 1

高度不是把所有子树高度相加,而是取最深的那一条路,所以用 max。

练习 C：最大根到叶路径和

节点标签都是数。求从当前节点到某个叶子的路径上,标签和的最大值。

函数 最大路径和(N)
// 节点 -> 数值
如果 N 是叶子
  返回 标签(N)
m <- -无穷
对 N 的每个子节点 E
  m <- max(m, ____)
返回 ____ + m
查看答案
函数 最大路径和(N)
// 节点 -> 数值
如果 N 是叶子
  返回 标签(N)
m <- -无穷
对 N 的每个子节点 E
  m <- max(m, 最大路径和(E))
返回 标签(N) + m

路径只能选择一个孩子继续往下,所以合并方式是 max,不是求和。

四、图递归：先建立已访问集合思维

A --- B --- D | | C --- E

图和树最大的不同在于：图里可能有环。递归访问图时,如果没有已访问集合 M,程序就可能在 A - B - A - B 之间无限循环。

图递归基本规则

进入顶点 s 后,立刻把 s 标记为已访问;之后只递归访问 t ∉ M 的邻居。

例 3：DFS 探索一个连通分量

过程 探索(G, s, M)
// 图 × 顶点 × 集合（会被修改） ->
将 s 加入 M
对 s 的每个邻居 t
  如果 t 不属于 M
    探索(G, t, M)

M 是会被修改的集合。调用结束后,M 中包含从 s 能到达的所有顶点。

练习 D：可达性

补全递归函数。它判断从 s 是否能到达 目标。

函数 可达(G, s, 目标, M)
// 图 × 顶点 × 顶点 × 集合 -> 布尔值
如果 s = 目标
  返回 ____
将 s 加入 M
对 s 的每个邻居 t
  如果 t 不属于 M
    如果 可达(G, t, 目标, M)
      返回 ____
返回 ____
查看答案
函数 可达(G, s, 目标, M)
// 图 × 顶点 × 顶点 × 集合 -> 布尔值
如果 s = 目标
  返回 真
将 s 加入 M
对 s 的每个邻居 t
  如果 t 不属于 M
    如果 可达(G, t, 目标, M)
      返回 真
返回 假

存在性问题的写法是：“只要找到一个成功分支,就立刻返回 真。”只有所有分支都失败时,才返回 假。

五、正式写之前,先做结构规划

结构规划不是额外步骤,它是防止递归题写乱的核心。下面这张表可以直接当草稿纸模板。

规划项树题怎么填图题怎么填
输入类型节点 或 树图 × 顶点,常再加一个 集合
输出类型整数、布尔值、列表、数值布尔值、整数,或直接修改集合
基准情形叶子、空孩子列表、标签满足目标已到目标、没有新邻居、顶点已访问
局部贡献当前节点是否计数、当前标签是多少、当前局部分数是多少标记当前顶点、当前顶点是否计数
递归对象每个孩子 E每个未访问邻居 t
合并方式加法、最大值、最小值、逻辑或、路径拼接加法、逻辑或、修改已访问集合

示范：从题目到伪代码

题目

函数 f(N) 输入一棵标签为数的树,返回“所有节点中,其孩子标签和”的最大值。叶子孩子标签和按 0 计。

输出最大值,所以返回 数值
当前贡献当前节点所有孩子标签之和 s
递归对象每个孩子 E
合并方式max(当前贡献, 每个子树最大值)
函数 f(N)
// 节点 -> 数值
s <- 0
m <- 0
对 N 的每个子节点 E
  s <- s + 标签(E)
  m <- max(m, f(E))
返回 max(s, m)

六、完整伪代码练习

这一组按难度递增。建议先只写结构规划,再写完整伪代码,最后打开答案对比。

  1. 练习 1：树中的最小标签

    写函数 最小标签(N),返回树中所有节点标签的最小值。

    参考答案
    函数 最小标签(N)
// 节点 -> 数值
m <- 标签(N)
对 N 的每个子节点 E
  m <- min(m, 最小标签(E))
返回 m
  2. 练习 2：树中恰有两个子节点的节点数

    写函数 两个子节点数(N),返回整棵树中子节点数恰好等于 2 的节点个数。

    参考答案
    函数 两个子节点数(N)
// 节点 -> 整数
计数 <- 0
如果 子节点个数(N) = 2
  计数 <- 1
对 N 的每个子节点 E
  计数 <- 计数 + 两个子节点数(E)
返回 计数
  3. 练习 3：树中是否存在标签 x

    写函数 是否包含(N, x),若树中有某个节点标签等于 x,返回 真,否则返回 假。

    参考答案
    函数 是否包含(N, x)
// 节点 × 标签 -> 布尔值
如果 标签(N) = x
  返回 真
对 N 的每个子节点 E
  如果 是否包含(E, x)
    返回 真
返回 假
  4. 练习 4：从根到目标的路径

    写函数 路径(N, x)。若存在标签 x,返回从根到该节点的标签列表;否则返回空列表。

    参考答案
    函数 路径(N, x)
// 节点 × 标签 -> 列表
如果 标签(N) = x
  返回 [x]
对 N 的每个子节点 E
  C <- 路径(E, x)
  如果 C 非空
    返回 [标签(N)] + C
返回 []

    这是“存在性 + 返回结构”的题。找到子树中的路径后,要把当前根节点接到前面。

  5. 练习 5：图中可达顶点数

    写递归过程 探索(G, s, M),使调用结束后 M 包含从 s 可达的所有顶点。再写函数 可达顶点数(G, s) 返回可达顶点个数。

    参考答案
    过程 探索(G, s, M)
// 图 × 顶点 × 集合（会被修改） ->
将 s 加入 M
对 s 的每个邻居 t
  如果 t 不属于 M
    探索(G, t, M)

函数 可达顶点数(G, s)
// 图 × 顶点 -> 整数
M <- 空集合()
探索(G, s, M)
返回 元素个数(M)
  6. 练习 6：图的连通分量数

    写函数 连通分量数(G),返回无向图的连通分量个数。

    参考答案
    函数 连通分量数(G)
// 图 -> 整数
M <- 空集合()
n <- 0
对 G 的每个顶点 s
  如果 s 不属于 M
    n <- n + 1
    探索(G, s, M)
返回 n

    每遇到一个还没访问过的顶点,就说明发现了一个新的连通分量,然后用 DFS 一次性标记整个分量。

  7. 练习 7：有限步可达

    写函数 限步路径(G, s, t, k, M),判断是否存在从 s 到 t 且长度不超过 k 的路径。可以假设路径不重复顶点。

    参考答案
    函数 限步路径(G, s, t, k, M)
// 图 × 顶点 × 顶点 × 整数 × 集合 -> 布尔值
如果 s = t
  返回 真
如果 k = 0
  返回 假
将 s 加入 M
对 s 的每个邻居 u
  如果 u 不属于 M
    如果 限步路径(G, u, t, k - 1, M)
      返回 真
返回 假

    这里 k 是递归规模。每走过一条边,就把 k 减 1。

  8. 练习 8：从树切换到图的思维

    解释为什么树递归通常不需要 M,而图递归通常需要 M。再写一句话说明 M 应该在什么时候更新。

    参考答案

    树天然不会沿着子节点方向回到父节点形成环,而且每个节点通常只有唯一父节点,所以沿子节点递归时不会无限绕圈。图则可能有环,例如 A-B-A,因此必须记录已经访问过的顶点。

    进入顶点后就应尽早把它加入 M,通常是在递归函数一开始、遍历邻居之前。

七、递归题高频错误清单

  • 把“当前节点的贡献”漏掉,只返回子树结果。
  • 在 for 循环第一轮就 返回,导致只处理了第一个孩子。
  • 该用 max 的题误用求和,尤其是高度和路径最优值。
  • 该用求和的题误用 max,尤其是节点计数。
  • 图递归忘记 M,或者标记太晚,导致重复访问。
  • 把“邻居”与“所有可达顶点”混淆。
  • 伪代码没写签名（signature）,白白丢掉容易拿的分。
  • 在图题里修改 G 本身,而题目只要求遍历。
  • 路径题找到子路径后忘记把当前节点接在前面。
  • 布尔存在性题在所有分支都失败后忘记返回 假。
最后的自检句

写完后问自己：如果当前节点是叶子会怎样？如果当前节点有 3 个孩子,会不会全部处理到？如果图里有环,会不会无限递归？返回值类型是不是和函数签名一致？
